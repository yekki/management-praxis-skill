## 实战案例：一家奶茶店要不要开第二家分店

以下是完整的分析过程演示。

### 第一步：描述问题

```
我开了一家奶茶店，生意还不错，在考虑要不要开第二家分店。
月营收 15 万，净利 3 万，开业一年半。
手上有 30 万现金，没有合伙人。
```

### 第二步：AI 诊断问题

AI 会结构化你的问题：

```
【问题定义】
单店奶茶品牌评估是否开设第二家分店——扩张决策。

【问题属性】
- 类型：战略（增长路径选择）
- 层次：公司级
- 紧迫度：中
- 关键约束：30万现金、无合伙人、单人管理
```

### 第三步：AI 推荐工具链

```
🔧 主工具：
1. BCG矩阵(1.4) — 评估现有业务的增长潜力和市场地位
2. 产品/市场扩展方格图(8.8) — 明确增长路径（开新店 vs 做新品 vs 其他）

🔩 辅助工具：
3. 本量利图(7.4) — 算清楚第二家店的盈亏平衡点

✅ 验证工具：
4. KT决策法(1.15) — 系统评估"开 vs 不开"的风险

【应用顺序】：BCG → 产品/市场扩展 → 本量利 → KT决策
```

你确认后，进入逐工具引导。

### 第四步：逐工具引导

#### 工具 1：BCG矩阵

AI 提供模板，你填入实际数据：

```
📊 BCG 定位分析

市场增长率：本地奶茶市场年增长 ≈ 15%（高）
相对市场份额：你在本区域的份额 ≈ 中等

→ 定位：⭐ 明星业务（高增长 / 中份额）
→ 含义：值得投资，但需要资金支持增长
```

#### 工具 2：产品/市场扩展方格图

```
📊 安索夫矩阵

              现有产品        新产品
现有市场    市场渗透 ←你在这   产品开发
            (提升单店营收)     (新品类)

新市场      市场开发 ←考虑这   多元化
            (开第二家店)       (风险最高)
```

AI 分析：你还没吃透现有市场（单店月营收 15 万有没有天花板？），就急着开新店，可能是**跳过了市场渗透直接做市场开发**。

#### 工具 3：本量利分析

```
📊 第二家店盈亏平衡

固定成本（月）：
  房租          ¥25,000
  人工（3人）    ¥18,000
  设备折旧       ¥3,000
  其他           ¥4,000
  合计          ¥50,000

变动成本率：约 40%（原料）
客单价：¥18
单杯变动成本：¥7.2

盈亏平衡点 = 50,000 / (18 - 7.2) = 4,630 杯/月 ≈ 155 杯/天

→ 你的第一家店日均多少杯？如果 > 200 杯，说明需求足够
→ 第二家选址区域的人流量能支撑 155 杯/天吗？
```

#### 工具 4：KT决策法

```
📊 KT 决策分析

              开第二家店    先做深单店
必须条件：
  30万够启动     ✅（刚好）    ✅
  不影响第一家    ⚠️ 精力分散   ✅

期望条件（加权）：
  收入增长(0.3)   4 → 1.2     2 → 0.6
  风险可控(0.3)   2 → 0.6     4 → 1.2
  管理简单(0.2)   1 → 0.2     4 → 0.8
  品牌提升(0.2)   3 → 0.6     2 → 0.4

总分              2.6          3.0 ✅
```

### 第五步：关键发现

| 发现 | 信心 | 来源 |
|------|------|------|
| 业务处于明星象限，有增长潜力 | B | BCG |
| 尚未吃透现有市场就扩张，风险偏高 | A | BCG + 安索夫 |
| 30万现金刚好够开店，无安全余量 | A | 本量利 |
| 单人管理两家店精力分散是最大风险 | A | KT决策 |
| "先做深单店"综合评分高于"开第二家" | B | KT决策 |

### 第六步：结论与建议

```
结论：现阶段不建议立即开第二家店

行动建议：
P0  先把第一家店营收从15万提到20万（验证天花板）
P0  建立标准化运营手册（为将来复制做准备）
P1  培养一名店长（解决"无合伙人"的管理瓶颈）
P1  6个月后重新评估，届时应有：
    - 第一家店稳定月净利 > 4万
    - 现金储备 > 50万（开店30万+半年安全垫20万）
    - 有合格店长可独立运营
P2  考察2-3个备选商圈，积累选址数据
```

### 第七步：生成报告

AI 自动生成两份报告：
- `reports/tea-shop-expansion.md` — Markdown 版
- `reports/tea-shop-expansion.html` — HTML 版（含可视化图表）

## 常用命令速查

| 你说 | AI 做什么 |
|------|----------|
| 描述一个管理问题 | 启动5阶段完整分析 |
| "快速分析：XXX" | 跳过追问，直接推荐工具和结论 |
| "换个工具" | 回到工具匹配阶段，重新推荐 |
| "跳过这个" | 跳过当前工具，进入下一个 |
| "深入分析" | 对当前工具结果做更深入展开 |
| "直接出报告" | 基于已有分析直接生成报告 |
| "用 SWOT 分析" | 指定使用某个工具 |

## 提示

1. **信息越具体，分析越准** — "月营收15万"比"生意还不错"好 10 倍
2. **不用担心选错工具** — AI 会根据你的问题自动匹配，你只需确认
3. **可以随时调整** — 分析过程中任何阶段都可以回退或跳过
4. **工具不是越多越好** — 3-5 个精准组合 > 10 个浅尝辄止
5. **报告是起点不是终点** — 用报告对齐团队认知，再做具体执行计划
