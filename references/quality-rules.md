# 质量规则

## 信心等级

| 等级 | 定义 | 标准 |
|------|------|------|
| A | 高度可信 | 3+ 个工具交叉验证一致 |
| B | 可靠 | 2 个工具支持 |
| C | 待验证 | 单一工具结论，需补充分析 |

## So What 检验

每个分析发现必须通过三层追问：
1. **So What?** — 这个发现意味着什么？
2. **Why does it matter?** — 为什么重要？
3. **What should we do?** — 应该怎么做？

如果三层追问后无法得出具体行动建议，该发现不进入报告。

## 反模式清单

| 反模式 | 描述 | 纠正 |
|--------|------|------|
| 工具堆砌 | 罗列5个以上工具但每个都浅尝辄止 | 精选3个深入应用 |
| 模板空转 | 填了模板但没有洞察 | 每个模板必须产出至少1条发现 |
| 结论先行 | 先有结论再找工具佐证 | 工具分析在前，结论在后 |
| 忽略矛盾 | 不同工具的矛盾结论被回避 | 矛盾点必须正面讨论 |
| 脱离场景 | 分析很专业但不贴合用户实际 | 每条建议必须考虑用户约束 |
